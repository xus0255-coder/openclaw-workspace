# 短剧制作一体化技能 (Short Drama Production Skill)

## 🎯 功能概述
整合短剧制作全流程所需的所有AI工具，提供从剧本分析到视频生成的标准化自动化工作流。

## 🎬 全流程阶段

```
📝 剧本分析  →  👤 角色设计  →  🏙️ 场景规划  →  🎞️ 分镜生成  →  🎬 视频制作
     ↓                ↓               ↓                ↓               ↓
  人物关系图       角色立绘         场景概念图         分镜故事板         成片视频
  场景清单         风格设定         环境氛围图         镜头描述           片段合集
  情感节奏图       服装道具图       地图位置图         动作说明           宣传物料
```

## 🛠️ 标准工作流

### Phase 1：剧本分析（使用 shortdrama-universal-analyzer 技能）
```
输入：剧本文本
输出：
  - 人物关系图谱
  - 场景清单（地点+时间+情绪）
  - 核心冲突节点
  - 关键视觉要素
```

### Phase 2：角色立绘生成（使用 libtv-skill）
```
输入：角色描述（来自剧本分析）
提示词模板：
  "角色描述，[风格], [表情状态], 上半身人物, 竖屏构图, 影视级光影, 无字幕水印"

质量标准：
  - 分辨率：768x1024（竖屏9:16）
  - 风格一致性：同剧角色保持视觉统一
  - 评分阈值：≥0.80
```

### Phase 3：场景概念图（使用 libtv-skill）
```
输入：场景描述（来自剧本分析）
提示词模板：
  "场景描述, [风格], [时间], 电影级构图, 无人物, 宽幅横屏, 专业摄影"

质量标准：
  - 分辨率：1920x1080（横屏16:9）
  - 布局：4场景2x2网格展示
  - 评分阈值：≥0.75
```

### Phase 4：分镜生成（使用 libtv-skill）
```
输入：关键情节节点
流程：
  1. 发送分镜指令到libtv（"帮我出一个9宫格分镜图"）
  2. 轮询等待生成完成
  3. 下载分镜图
  4. 质量评估（≥0.75）
```

### Phase 5：视频制作（使用 libtv-skill）
```
输入：分镜图 + 场景描述
流程：
  1. 上传参考分镜图
  2. 发送"根据分镜生成视频"指令
  3. 长轮询（每8秒查询，最长3分钟）
  4. 下载成片
```

## 🚀 快速使用

### 一键全流程（标准模式）
```
@短剧制作一体化 全流程 --script "剧本路径或内容" --style "都市现实"
```

### 单阶段执行
```
@短剧制作一体化 角色立绘 --characters "外卖小哥李明,28岁,帅气,穿黄色制服"
@短剧制作一体化 场景图 --scenes "外卖站点,早晨,忙碌氛围"
@短剧制作一体化 分镜 --plot "高潮场景描述"
@短剧制作一体化 视频 --storyboard "分镜图路径"
```

### 批量生成
```
@短剧制作一体化 批量角色 --script "剧本路径"
# 自动识别所有主要角色，批量生成立绘，质量评估后输出达标图片
```

## 📊 质量控制节点

| 阶段 | 评估指标 | 合格阈值 | 不合格处理 |
|------|----------|----------|------------|
| 角色立绘 | 清晰度+人物比例+风格 | ≥0.80 | 自动重试，最多3次 |
| 场景概念图 | 构图+氛围+细节 | ≥0.75 | 调整提示词重试 |
| 分镜图 | 故事连贯性+视觉清晰 | ≥0.75 | 重新生成 |
| 视频片段 | 流畅度+画质+内容 | ≥0.70 | 人工审核 |

## ⚙️ 配置参数

```yaml
drama_production:
  default_style: "都市现实"          # 默认风格
  character_ratio: "9:16"            # 角色图比例
  scene_ratio: "16:9"                # 场景图比例
  quality_auto_retry: true           # 质量不达标自动重试
  max_retries: 3                     # 最大重试次数
  output_base_dir: "./短剧制作输出/" # 输出目录
  libtv_session_reuse: true          # 复用libtv会话
  parallel_generation: true          # 并行生成多个资产
```
