# 订单截图 → 目录表 完整工作流

从电商订单截图（京东/淘宝）提取结构化信息，生成标准化目录表。

## 输入要求

截图命名格式（按时间排序）：
- `Screenshot_YYYYMMDD_HHMMSS_com.jingdong.app.mall.jpg`（京东）
- `Screenshot_YYYYMMDD_HHMMSS_com.taobao.taobao.jpg`（淘宝）

放入目录：`F:/我的文件夹/正式文件/购物/`

## 完整工作流

### 步骤1：安装依赖

```bash
pip install rapidocr_onnxruntime
pip install python-docx
pip install reportlab
```

> rapidocr 首次安装会自动下载模型（~13MB），之后完全离线运行。

### 步骤2：OCR批量提取

运行 `batch_ocr_v2.py`：
- 输入：`F:/我的文件夹/正式文件/购物/*.jpg`
- 输出：`F:/我的文件夹/正式文件/购物/ocr_results.json`
- 提取字段：商品名称、数量、总价、购买时间、来源（京东/淘宝）

**关键提取逻辑**：
- 商品名：定位店铺行（含"旗舰店"/"专卖店"）之后、价格行（"到手"/"合计"/"实付款"）之前的文本
- 总价正则：`合计`、`到手`、`实付款\s*[￥¥]?\s*(\d+...)`、淘宝备选最后一个`￥数字`
- 时间正则：`下单时间`、`支付时间`、淘宝`交易成功\s+日期`
- 数量正则：`数量[×x]\s*(\d+)`

### 步骤3：生成Word目录表

运行 `generate_tables_v2.py`，生成3张Word表：

| 文件 | 内容 |
|------|------|
| `目录表_儿童用品和书籍.docx` | 儿童/书籍类，约36条 |
| `目录表_其他商品.docx` | 其余商品，约135条 |
| `目录表_全部商品.docx` | 全部171条 |

**Word表格式**：
- 列：**页码 | 商品名称 | 数量 | 总价 | 购买时间 | 来源**
- 无文件名列、无单价列
- 总价纯数字（无¥符号），宋体
- 末行汇总：合计 | 共X条 | 空 | 累计金额 | 空 | 空

### 步骤4：生成PDF目录表（含页码）

运行 `generate_pdf.py`，生成带页码的PDF：

| 文件 | 内容 |
|------|------|
| `目录表_儿童用品和书籍.pdf` | 儿童/书籍类 |
| `目录表_其他商品.pdf` | 其余商品 |
| `目录表_全部商品.pdf` | 全部记录 |

**PDF格式**：
- 字体：SimSun（宋体），`C:/Windows/Fonts/simsun.ttc`
- 页码：底部居中显示"第 X 页"
- 表头深绿色（`#2C5AA0`），数据行交替底色
- 汇总行浅橙色背景（`#FFF3E0`），金额红色高亮

## 注意事项

- 商品名称因截图UI截断可能不完整（如"【抖音同款】漫"），需人工核对补充
- 淘宝截图金额在"实付款"下一行，需正则跨行匹配
- 若无金额记录，检查`ocr_results.json`并手动补充
- 每次新增截图后重新运行全部3个脚本
